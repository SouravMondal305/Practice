# Code Question 1


### You are given:

* An integer array **population** of size **n**, where **population[i]** is the number of inhabitants in the **iᵗʰ city**.
* A binary string **unit** of length **n**, where

  * **unit[i] = '1'** means city **i** has a security unit
  * **unit[i] = '0'** means it does not.

---

# Relocation Rule

* A security unit at city **i (where i > 1)** can be moved **one step to the left to city i-1**.
* Each unit can be **moved at most once**.
* If moved, **city i loses its unit and city i-1 gains one**.
* **City 1’s security unit cannot be moved further left.**

---

# Objective

A city is **protected** if it has a security unit after all relocations.

Determine the **maximum population that can be protected** by optimally relocating the security units.

---

# Note

The problem uses **1-based indexing**, meaning:

* city indices
* the population array
* the unit binary string

all start from **1** and go up to **n**, with each city having a corresponding entry in both the array and the string.

---

# Example

```
n = 5
population = [10, 5, 8, 9, 6]
unit = "01101"
```

### Relocations

| Relocated Index (j): (j → j-1) | Previous Configuration | New Configuration | Safe Inhabitants |
| ------------------------------ | ---------------------- | ----------------- | ---------------- |
| 2                              | 01101                  | 10101             | 10 + 8 + 6 = 24  |
| 5                              | 10101                  | 10110             | 10 + 8 + 9 = 27  |

Thus **27 is the maximum number of inhabitants that can be protected**.

No other relocation strategy can yield a higher safe population than **27**.

Hence answer = **27**.

---

# Function Description

Complete the function **moveUnits**.

### Parameters

```
int population[n]   → population of each city
string unit         → initial configuration of units
```

### Returns

```
int → the maximum population that can be safe after moving some security units to their left
```

---

# Constraints

```
1 ≤ n ≤ 10^5
1 ≤ population[i] ≤ 10^4
unit is a binary string consisting of '0's and '1's
```

---

# Sample Input 0

```
6
20
10
9
30
20
19
011011
```

Meaning

```
population = [20,10,9,30,20,19]
unit = "011011"
```

---

# Sample Output 0

```
80
```

---

# Explanation

| Relocated Index (j) | Previous | New    | Safe Inhabitants       |
| ------------------- | -------- | ------ | ---------------------- |
| 2                   | 011011   | 101011 | 20 + 9 + 20 + 19 = 68  |
| 3                   | 101011   | 110011 | 20 + 10 + 20 + 19 = 69 |
| 5                   | 110011   | 110101 | 20 + 10 + 30 + 19 = 79 |
| 6                   | 110101   | 110110 | 20 + 10 + 30 + 20 = 80 |

Thus **80 is the maximum number of inhabitants that can be protected**.

No other relocation strategy yields a higher safe population.
